\chapter{绪论}
\label{cha:introduction}

\section{研究背景}
\label{sec:background}

企业内部的知识服务正加速向生成式人工智能迁移，文档问答是其中已有大量工程实践的典型场景之一\cite{feuerriegel2024generative}。然而，仅依赖参数记忆的生成模型在封闭域问答任务中难以避免幻觉问题，其输出可能包含事实错误且往往无从溯源\cite{zhang2025hallucination}。当问题涉及制度条款、财务数据或内部流程时，此类偏差将直接影响合规判断与审计结论的可靠性。

为缓解幻觉与输出无法溯源的问题，学界提出检索增强生成（Retrieval-Augmented Generation, RAG）范式：在生成前从外部文档库检索候选证据，再依据证据组织回答，使生成结果有据可查，而非单纯依赖模型参数中的隐式知识。这也是 RAG 在企业文档场景中得以落地的核心原因。

企业应用场景对问答系统的要求远不止“答得对”。监管合规要求系统具备权限控制与内容安全能力\cite{genai_measures2023}，企业文档本身又往往篇幅长、版式复杂、持续更新。更根本的问题在于，O'Leary 指出企业知识服务的有效运行依赖知识组织、权限治理与业务流程的协同\cite{oleary2024km}。这意味着文档问答系统面对的挑战，从来就不只是回答质量，而是能否在权限约束与文档持续演化的条件下稳定输出可信结果。

为满足上述需求，工程实现层面还需应对三类彼此关联的问题。一是企业文档体量大、更新频繁，入库链路一旦不稳定，系统就无法持续提供可检索的知识。第二个问题是权限边界。企业文档的可见范围由角色、组织与公开属性共同约束，若检索边界与访问控制边界不一致，越权召回就不可避免。第三类问题更隐蔽：企业用户的问询往往是连续追问，跨轮次的证据一致性和来源可追溯性，直接决定系统在实际业务场景中是否值得信任。

\section{国内外研究现状}
\label{sec:related_work}

本节分别先讨论 RAG 检索链路的演进与企业知识场景中的工程化实践，随后归纳企业私有文档问答仍待解决的关键问题。

\subsection{RAG 技术的发展}

RAG 的技术基础来自两类检索路径的汇合。语义检索一侧，向量空间模型\cite{salton1975vectorspace}奠定了稠密表示的早期框架，稠密段落检索（Dense Passage Retrieval, DPR）\cite{karpukhin2020dpr}则将其推进到神经网络时代；词法检索一侧，Best Matching 25（BM25）\cite{robertson2009bm25}至今仍是强基线。Facebook AI Similarity Search（FAISS）\cite{johnson2021faiss}解决了大规模相似度搜索的工程瓶颈，Lewis 等人在此基础上将检索器与生成器统一为端到端链路，正式确立了“先检索、后生成”的基本范式\cite{lewis2020rag}。

随着研究推进，关注点逐渐从静态一次性检索转向更精细的动态组织方式。假设性文档嵌入（Hypothetical Document Embeddings, HyDE）以查询改写为切入点，通过构造假设性文档提升向量匹配质量\cite{gao2023hyde}；前瞻式主动检索增强生成（Forward-Looking Active Retrieval Augmented Generation, FLARE）将检索时机的判断嵌入生成过程，使外部知识得以按需补充\cite{jiang2023flare}。自反思检索增强生成（Self-RAG）的方案更为激进，将检索决策与自我校验一并纳入训练目标，试图使模型自主管理检索行为\cite{asai2024selfrag}。比较研究的结果也印证了这一趋势，RAG 系统的效果很大程度上取决于检索器、重排序器与上下文组织方式的协同配置，而非单一模型的能力\cite{rahman2025comparative}。

不过，上述研究有一个共同的隐含前提：知识源已经完成清洗、切分与索引，评测对象是静态语料库\cite{lewis2020rag,karpukhin2020dpr,jiang2023flare,asai2024selfrag}。该前提在学术基准评测中成立，但在企业私有文档场景中通常不成立。文档上传中断后的状态恢复、增量更新一致性以及权限边界在检索阶段的落实，均不在主流 RAG 研究的讨论范围内，而这些问题直接决定了系统能否进入持续可用的实际部署状态。

\subsection{企业知识场景中的工程化研究}

企业知识场景中的工程化研究，出发点往往不是“用什么模型”，而是“知识在进入模型之前如何被组织”。O'Leary 的系列工作集中体现了这一立场：他讨论了企业知识库与提示资产的管理关系\cite{oleary2024km}，也分析了大语言模型在企业系统设计中的边界与约束\cite{oleary2024enterprise}。两项研究共同表明，文档组织、版本管理与治理基础是企业知识问答系统得以有效运行的结构性前提，而非可选的附加能力。

在行业应用层面，RAG 已被用于建筑工程知识管理\cite{wang2025buildingkm}、工业设备运维\cite{rubio2025industrialkm}及政策文本检索等场景\cite{policyrag2025}。但其实验设置均以预设的、相对稳定的文档集合为前提，未涉及多组织环境下权限边界的动态变化问题。

在通用框架层面，RAGLAB 提供了模块化实验基准\cite{zhang2024raglab}，FlexRAG 支持多种检索运行模式\cite{zhang2025flexrag}，RAGVA 则强调工程实践中的可替换性与可观测性\cite{vogelsang2025ragva}。但上述框架的关注点仍在组件装配与运行模式，对企业私有文档场景中文档可靠入库、权限前置过滤与多轮证据边界维持等问题，尚未给出直接回答。

\subsection{现有研究的不足}

若回到企业私有文档问答这一具体场景，现有研究仍留下三处较为明显的缺口。

第一，文档接入可靠性在现有研究中几乎是一个被跳过的问题。现有工程化研究和系统框架的重点落在检索链路组织、组件装配与问答效果上\cite{zhang2025flexrag,vogelsang2025ragva,wang2025buildingkm,rubio2025industrialkm}，即使面向实际应用的系统，也倾向于将文档接入视为一次性的离线预处理步骤。在持续更新的企业私有文档库中，上传中断的恢复机制与前台请求和后台入库的解耦设计不可或缺；若这两类问题未能解决，文档将无法稳定进入可检索状态，后续的检索质量评估也失去基础。

第二，权限控制尚未真正进入候选集生成阶段。已有综述已将越权召回列为可信 RAG 的核心风险\cite{ni2025trustworthyrag}，但现有领域系统多在单域或预设文档集合上验证\cite{wang2025buildingkm,rubio2025industrialkm,policyrag2025}，权限过滤实际上是在召回完成后才介入的后置裁剪。这意味着未授权文档仍可进入向量近邻搜索与重排序的完整流程，不仅产生额外计算开销，也使检索边界与访问控制边界在根本上无法对齐。

第三，多轮会话中的证据边界维持缺少专门设计。语言模型对长上下文中部内容的利用本身就不稳定\cite{liu2024lostmiddle}，而 FLARE、Self-RAG 等动态检索方法虽然缓解了单轮信息缺失的问题\cite{jiang2023flare,asai2024selfrag}，却没有面向企业文档问答的特定需求，例如历史对话如何约束当前检索范围、跨轮来源如何保持追溯一致性。在企业实际使用场景中，连续追问是常见交互模式；跨轮证据边界与来源追溯机制的缺失，将直接削弱系统在多轮对话中的可信度与可审计性。

现有工作分别讨论了检索优化、知识治理或系统搭建中的局部问题，但当文档持续更新、权限动态变化且问答跨轮进行时，系统如何同时保证可靠接入、授权一致和证据边界稳定，仍缺少统一回答。

\section{研究内容与主要贡献}
\label{sec:research_content}

本文围绕企业私有文档问答场景，分别就文档接入、权限控制、多轮会话与隐私治理四类工程问题提出解决方案，相应工作可概括为以下四点。
\begin{enumerate}[label=（\arabic*）]
    \item 针对大文件入库链路长、中断后难以恢复的问题，设计了断点续传与异步入库机制。核心思路是将上传确认与后台解析入库解耦，通过 Redis 位图实现分片级幂等，通过 Kafka 消息队列将长任务移出在线请求路径，使文档接入从同步阻塞流程转变为可恢复的后台流水线。第五章的测试结果显示，20 份大文档的续传成功率与 MD5 一致率均为 100\%，200 份样本均完成上传、解析与索引。
    
    \item 针对权限过滤在召回之后才介入的结构性问题，提出基于角色的访问控制（Role-Based Access Control, RBAC）与层级组织标签的检索前置权限过滤方案，将访问控制条件内嵌于候选集生成阶段。该方案把用户身份、组织范围与文档公开属性编码为 Elasticsearch 的 \texttt{filter} 子句，使权限判定在向量召回与词法匹配阶段即完成，避免未授权文档进入重排序与上下文装配流程。权限矩阵测试覆盖 30 组样本，越权返回率为 0。
    
    \item 针对多轮问答中证据边界随轮次漂移、历史上下文可能放宽权限判定的问题，构建了轮次分类驱动的会话状态控制机制。该机制通过 \texttt{NEW\_TOPIC}、\texttt{FOLLOW\_UP}、\texttt{META}、\texttt{CLARIFICATION} 四类轮次判定，动态选择 \texttt{SELECTIVE}、\texttt{RAW\_RECENT}、\texttt{DISABLED} 三种历史注入策略；通过工作记忆维护活动文档集合，限定后续检索范围；通过会话级引用编号映射，维持跨轮来源追溯一致性。对照实验显示，选择性历史策略在语义通过率与长程历史回溯两项指标上均优于近轮拼接和无历史基线，引用标记通过率为 100\%。
    
    \item 针对企业文档中敏感信息在索引、外发与日志阶段面临不同风险等级的问题，设计了分块级隐私治理机制。该机制为每个文本块维护原始文本、安全文本与归一化文本三个视图，并在分块级记录敏感级别、受限标记、外发许可与敏感类型等元数据；索引阶段使用安全文本，外发阶段校验 \texttt{externalModelAllowed} 标志，日志阶段截断为最小预览。敏感信息测试覆盖身份证、手机号、邮箱、银行卡、应用程序接口密钥（API Key）与受限关键词六类场景共 48 组样本，泄漏率为 0。
\end{enumerate}

\section{论文结构安排}
\label{sec:arrangement}

全文共六章。

第一章阐述研究背景与动机，梳理国内外相关工作的进展与不足，并界定本文的研究问题；

第二章分析系统功能需求与非功能约束；

第三章阐述系统总体架构，重点说明权限模型与多租户隔离策略、文档接入链路、混合检索机制及会话状态控制方案，这四部分构成本文的核心设计；

第四章说明核心模块的实现细节，包括身份认证、分片上传与异步入库、混合检索、提示词（Prompt）装配、会话管理与流式问答；

第五章在真实环境下开展功能验证与性能测试，并通过对照实验评估关键设计决策的有效性；

第六章总结本文工作，分析局限，并提出后续改进方向。
