\chapter{总结与展望}
\label{cha:conclusion}

\section{全文总结}

\subsection{研究总结}

本文针对企业私有文档知识服务的实际需求，以检索增强生成为核心架构，围绕权限受控的文档知识闭环这一核心问题，设计并实现了涵盖接入、入库、检索与问答全流程的原型系统。本文的主要贡献在于系统性地解决了企业私有文档问答场景中的工程化问题，将权限控制、异步处理、检索编排与多轮会话管理整合为统一架构，并通过实验验证了核心设计决策的有效性。

功能测试表明，系统在单机部署条件下完成了从文档接入、知识入库到权限受控问答的全流程验证；性能测试结果显示，在当前样本和部署边界内，检索效果、并发响应能力与端到端时延均达到预设目标。对照实验进一步表明，在本次测试设置下，混合检索策略与多轮会话历史管理机制分别在 Recall@5、Recall@1 与多轮语义通过率等指标上优于相应基线。上述结果提示，在企业私有文档场景中，检索编排与会话管理等工程设计会直接影响系统的问答表现，系统效果不能简单归因于底层语言模型本身。

\subsection{局限性分析}

本文工作存在以下几方面局限。一是实验规模仍然有限，当前评测主要基于 200 份文档、400 条检索查询和受控会话样本展开，向量化效率与检索排序一致性尚未经更大规模场景验证。二是安全性测试仍不充分，虽然论文已对权限边界、隐私治理和对抗样本进行了初步检查，但对提示注入等复杂对抗性输入尚未形成系统性的分类型评估。三是运行环境仍以受控条件下的单机部署为主，相关结论尚不能直接外推到长期运行和规模化部署场景。

因此，本文工作定位于面向受控场景的可行性验证，所提架构在有限工程约束下的有效性已得到初步验证，但距规模化工程部署仍有差距。

\section{后续展望}

近期最迫切的工作是建立系统性评测基准。当前评测集规模有限、覆盖场景单一，难以支撑后续检索策略的可复现比较。未来需要围绕文档类型、权限层级与多轮问答行为构建更稳定的标注数据与评价方法，使检索优化从经验调参转向可验证的比较研究。在此基础上，查询改写、重排序、分块粒度等关键参数的系统性消融分析，以及多轮对话中语义漂移的抑制机制，都是值得继续深入的方向。

从工程演进角度看，当前原型距生产级部署仍有差距。可观测性、容错恢复与索引动态维护等工程基础能力，是系统走向规模化运维的前提条件。只有在长期运行约束下进一步验证这些能力，本文提出的架构才能继续评估其在真实企业环境中的稳定性与可维护性。

在研究方向上，多模态文档理解是当前系统的自然延伸。如何对图文混排、手写标注与复杂版式档案进行结构化理解，是后续扩展面临的核心挑战。更关键的是，多模态能力的引入要求将图像区域、表格单元与文本片段统一纳入权限过滤与证据溯源体系，这对现有架构的扩展性提出了实质性要求。上述扩展将使系统具备处理企业级复杂档案的基础能力。
